minimal-blog Post 增加 / 发布 / 删除 数据流图

左边是后台管理链路，右边是公开前台静态站链路。重点不是“前端直接查数据库”，而是：后台列表页来自 SQLite，公开前台来自 Markdown + build 后的 dist HTML。

新增 / Save Draft

  • editor.js 读取表单并组装 content_md
  • POST /api/posts/draft 写入 SQLite posts
  • 返回 /admin/posts 后，EJS 重新查库渲染列表

Publish

  • 先保存 draft，再调用 publish API
  • DB 更新 status/content_html/published_at
  • 写 site/content/posts/*.md，再 build 生成 dist/posts/*.html

Delete

  • 先删 SQLite 记录
  • 再删 markdown 源文件和 dist HTML
  • 后台当前页用 row.remove() 立即同步视觉结果
